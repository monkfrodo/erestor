    **Frontend (web/src/services/push.ts):**

    1. Export `requestPushPermission()`:
       - Check if `'serviceWorker' in navigator && 'PushManager' in window`
       - If not supported: return { supported: false }
       - Request Notification.requestPermission()
       - If granted: call subscribePush()
       - If denied: return { granted: false }

    2. Export `subscribePush()`:
       - Get service worker registration via `navigator.serviceWorker.ready`
       - Call `registration.pushManager.subscribe({ userVisibleOnly: true, applicationServerKey: urlBase64ToUint8Array(VAPID_PUBLIC_KEY) })`
       - Extract subscription JSON (endpoint + keys.auth + keys.p256dh)
       - POST to backend `/v1/webpush/subscribe` via apiFetch with subscription data
       - Return success/failure

    3. Export `unsubscribePush()`:
       - Get current subscription via `registration.pushManager.getSubscription()`
       - If exists: call subscription.unsubscribe()
       - DELETE to backend `/v1/webpush/subscribe` with subscription data

    4. Helper `urlBase64ToUint8Array(base64String)` for VAPID key conversion.

    5. Integration in page.tsx: After first user interaction (e.g., first chat message sent or first poll responded), call `requestPushPermission()`. Do NOT request on page load per user decision. Store permission state in localStorage to avoid re-prompting.

    **Backend (~/claude-sync/produtividade/):**

    6. Check Python version compatibility. If pywebpush requires Python 3.10+ and server has 3.9, use `py_vapid` + `http_ece` + `requests` directly for push sending. Otherwise install `pywebpush`.

    7. Create `api/routers/webpush.py`:
       - Router with Depends(verify_token) at router level
       - POST `/webpush/subscribe`: accepts { endpoint, keys: { auth, p256dh } }, stores in devices.json under "web" array, deduplicates by endpoint
       - DELETE `/webpush/subscribe`: removes subscription by endpoint
       - Helper `send_web_push(title, body, tag, actions=None, payload=None)`: loads all web subscriptions from devices.json, sends push to each using pywebpush (or manual implementation). VAPID private key from env var `VAPID_PRIVATE_KEY`, claims email from env var `VAPID_EMAIL`.
       - Handle expired/invalid subscriptions: catch WebPushException with 410 status, remove stale subscription

    8. Update `api/main.py`: import and include webpush router with prefix `/v1`.

    9. Update `api/deps.py` if needed: add query parameter token support for SSE endpoint (check if `verify_token` already accepts query param `token`; if not, add fallback to `request.query_params.get("token")` for EventSource compatibility).

    10. Update `api/routers/events.py`:
        - In the poll scheduler (where send_push/APNs is called), add `send_web_push()` call alongside it
        - For energy polls: `send_web_push(title="Energia", body="Como esta sua energia?", tag="poll_energy", actions=[{action:"ENERGY_1",title:"1"},{action:"ENERGY_2",title:"2"},...], payload={poll_id})`
        - For quality polls: `send_web_push(title="Bloco", body="Como foi o bloco?", tag="poll_quality", actions=[...], payload={poll_id})`

        - For daily synthesis: `send_web_push(title="Sintese", body="Sintese do dia disponivel", tag="synthesis")`
        - Import send_web_push at top of events.py
