       ```
       npm install -D vitest @testing-library/react @testing-library/jest-dom jsdom @vitejs/plugin-react
       ```

    3. Create `web/vitest.config.ts` configuring Vitest for Next.js (jsdom environment, react plugin, path aliases matching tsconfig).

    4. Create `web/src/app/globals.css` with Tailwind v4 `@import "tailwindcss"` and `@theme` block. Define ALL DS CSS custom properties mapping 1:1 from DesignSystem.swift (see interfaces). Set `body` background to `var(--ds-bg)`, color to `var(--ds-text)`. Import Google Fonts for IBM Plex Mono (weights 300,400,500) and Inter (weights 300,400,500).

    5. Create `web/src/lib/ds.ts` exporting DS constants as a TypeScript object for programmatic access (colors as hex strings, CSS var references).

    6. Create `web/src/app/manifest.ts` using Next.js MetadataRoute.Manifest:
       - name: "Erestor", short_name: "Erestor", display: "standalone"
       - background_color: "#1a1816" (DS.bg), theme_color: "#1e1c1a" (DS.surface)
       - icons: 192x192 and 512x512 PNGs
       - start_url: "/"

    7. Create placeholder PWA icons (`web/public/icon-192x192.png`, `web/public/icon-512x512.png`) -- generate simple dark square icons with "E" letter using a canvas script or copy from existing assets.

    8. Create `web/public/sw.js` -- push-only service worker handling `push` and `notificationclick` events (see RESEARCH.md code example). Include action button handling that POSTs to `/api/poll-respond` proxy route, and focus/open PWA on notification body click.

    9. Create `web/src/app/layout.tsx`:
       - Set metadata (title: "Erestor", description, themeColor)
       - Register service worker on mount via useEffect in a client component wrapper
       - Apply font classes (IBM Plex Mono, Inter) via Google Fonts next/font or link tags
       - Set viewport meta for PWA (width=device-width, initial-scale=1, viewport-fit=cover)

    10. Create `web/src/services/api.ts`:
        - Export `API_BASE` from env var `NEXT_PUBLIC_API_URL` (default: "http://localhost:8766")
        - Export `getToken()` reading from env var `NEXT_PUBLIC_API_TOKEN`
        - Export typed `apiFetch(path, options)` wrapper that adds Authorization header and parses JSON response with ApiResponse typing

    11. Create `web/src/services/sse.ts`:
        - SSEManager class with connect/disconnect methods
        - Connect to `${API_BASE}/v1/events/stream?token=${token}`
        - Listen for event types: context_update, poll_energy, poll_quality, gate_alert, heartbeat
        - On context_update: call `useContextStore.getState().update(data)`
        - On poll_energy/poll_quality: call `usePollStore.getState().addPoll(data)`
        - On gate_alert: call `usePollStore.getState().addGate(data)`
        - On heartbeat: reset retry delay to 3000ms
        - On error: close, reconnect with exponential backoff (3s -> 6s -> 12s -> 24s -> 30s cap)
        - Export singleton instance

    12. Create Zustand stores:
        - `contextStore.ts`: currentEvent, timer, tasks, nextEvent, update(data) action
        - `chatStore.ts`: messages array, isStreaming flag, addMessage, appendToken, finishStreaming actions
        - `pollStore.ts`: activePolls array, activeGates array, addPoll, addGate, removePoll, removeGate actions

    13. Create `web/.env.example`:
        ```
        NEXT_PUBLIC_API_URL=http://localhost:8766
        NEXT_PUBLIC_API_TOKEN=your_token_here
        NEXT_PUBLIC_VAPID_PUBLIC_KEY=
        ```

    14. Create tests:
        - `web/src/__tests__/manifest.test.ts`: import manifest function, assert it returns correct name, display, background_color, icons array
        - `web/src/__tests__/sse.test.ts`: test SSEManager reconnection logic (mock EventSource), test that store update functions are called on events
