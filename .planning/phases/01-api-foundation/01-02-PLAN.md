    Create `api/routers/chat.py`:
    - Define `ChatRequest(BaseModel)` with `message: str` (min_length=1)
    - `POST /chat/stream` endpoint with `response_class=EventSourceResponse`
    - Use FastAPI native SSE: `from fastapi.sse import EventSourceResponse, ServerSentEvent`
    - Implementation approach (pragmatic, per RESEARCH.md recommendation): use `call_claude` from `erestor.claude` via `asyncio.to_thread` -- this gives the full response in one chunk. Yield it as a single "message" SSE event, then yield a "done" event. True token-by-token streaming is a Phase 2 optimization.
    - The async generator pattern:
      ```python
      async def generate():
          response = await asyncio.to_thread(call_claude, req.message)
          yield ServerSentEvent(data=json.dumps({"text": response}), event="message")
          yield ServerSentEvent(data=json.dumps({"done": True}), event="done")
      return EventSourceResponse(generate())
      ```
    - If `call_claude` raises an exception, yield an "error" SSE event with the error message
    - Router with `Depends(verify_token)` at router level

    Update `api/main.py`:
    - Add `from api.routers import chat` and `app.include_router(chat.router, prefix="/v1")`

    Create `tests/test_chat_stream.py`:
    - Mock `erestor.claude.call_claude` to return a fixed string "Test response"
    - Test SSE response format: verify content-type is text/event-stream
    - Parse SSE events from response body -- check for "message" event with text and "done" event
    - Test auth enforcement
    - Test validation (empty message)
    - Use `httpx` stream reading or parse raw SSE text from response body

    NOTE: FastAPI TestClient with SSE requires reading the full response body and parsing SSE format manually (splitting on "event:" and "data:" lines). The httpx client should use `follow_redirects=True`.
